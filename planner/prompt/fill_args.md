# 参数填充说明（planner.arg_fill）

你将收到一个 JSON 对象，包含字段：

- `task`: 用户的自然语言任务描述。
- `skill`: 技能的简要信息：
  - `id`: 技能标识。
  - `selector`: 该技能对应控件的主选择器（用于理解模块大致位置）。
  - `by_text`: 若干与该技能相关的可见文案（例如“预订酒店 目的地/酒店名称 入住…”）。
- `args_schema`: 参数规格列表，每个元素包含：
  - `name`: 参数名（例如 `destination`、`check_in_date`）。
  - `type`: 参数类型（`string` / `integer` / `boolean` / 其他）。
  - `description`: 参数含义的自然语言说明。
  - `required`: 是否为必填参数（布尔值）。

你的目标：**根据 `task` 与 `args_schema`，为每个参数推断一个合理的取值**，并以 JSON 对象形式返回。

---

## 输出要求

- 必须只输出一个 JSON 对象，不要包含额外文字、说明或 Markdown。
- 键必须是参数名，值是你推断出的取值，例如：

```json
{
  "destination": "上海",
  "check_in_date": "11月16日",
  "check_out_date": "11月17日",
  "rooms": 2
}
```

- 对于 `required=true` 的参数：
  - 尽量根据任务中的信息给出明确值；
  - 若任务确实没有提供任何线索，只能在上下文中合理猜测一个“默认值”，但不要胡乱编造。
- 对于 `required=false` 的参数：
  - 如果任务中没有给出有用信息，可以直接省略这个字段。

---

## 推断规则（建议）

- 尽量直接使用任务中出现的具体值：
  - 城市、目的地名称；
  - 日期（例如“11月16日”、“2024-05-01”）；
  - 数量（房间数、人数、晚数等）。
- 日期、数量等字段的格式应尽量贴近任务中的写法：
  - 任务里是“11月16日”，参数中也使用“11月16日”；
  - 避免随意转换为完全不同格式，除非说明中有强约束。
- 布尔值参数：
  - 按说明判断是否需要启用某些选项（例如“只看可免费取消”）。

输出中不要加入任何额外解释，只返回参数 JSON 对象。
